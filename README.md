# ✍️ AI 프롬프트 디자이너 (Prompt Engineering Designer)

> **"좋은 질문이 좋은 AI 결과를 만듭니다."**  
> AI 프롬프트 디자이너는 중학생부터 일반 사용자까지 올바르고 정밀한 질문 설계 법칙을 배울 수 있는 **교육용 프롬프트 설계 플랫폼**입니다.

이 어플리케이션은 사용자가 AI에게 내리는 프롬프트를 구성 요소별(역할, 상황, 지시, 출력 형식, 조건)로 정교하게 커스텀 조립하며, 실시간 완성도 채점 및 모의 시뮬레이션을 통해 더 나은 프롬프트 설계 원칙을 이해하도록 유도합니다.

---

## ✨ 주요 제공 기능

1. **8단계식 마법사형 프롬프트 조립 시스템 (Stepper)**
   - **모드 선택**: 초급 (질문 힌트 지원), 중급 (직접 요소를 기입), 고급 (서브 역할, 상세 가독 규칙 추가)
   - **목적 선택**: 보고서, 글쓰기, 코드 검토, 공부법 등 12가지 카테고리 지원
   - **맥락 및 과제**: 제목, 당면과제, 최종 목표, 타겟 독자(Audience), 상황(Context) 구체화
   - **AI 역할 설정**: 친절한 코치, 비판적 분석 연구원 등 9종의 전문 페르소나 적용
   - **작업 및 정보**: 구체적 지시 문장 작성 및 참조 가능한 참고 자료 연계
   - **출력 규격**: 일반 문장, 리스트, 표 형식, 목차 전개 구조 강제
   - **조건 및 제약**: 포함 단어, 제외 항목(우회 방어), 발표 분량 제어

2. **실시간 프롬프트 컴파일러 & 정량 분석기 (Live Compiler)**
   - 입력 영역 변경에 따른 자동 조합 뷰어 및 복사
   - 글자 수, 단어 수, 문장 수, 예상 읽기 시간 및 토큰 수(Token Count)의 실시간 추산
   - 100점 만점 기준의 가독/지시 완성도 자가 채점 시스템 제공 및 부족 항목 자동 피드백

3. **로컬 모의 AI 실행기 (Mock Simulation) & 충돌 감지**
   - **레이아웃 예상**: 지정 포맷에 근거한 예상 응답 전개도 제공
   - **상치 조건 감지**: "매우 짧게"와 "자세히 설명"과 같이 서로 충돌하는 조건을 찾아내어 1클릭 해결 솔루션 제시
   - **모호한 문장 교정**: "멋지게 써줘" 등 의미가 불분명한 부사를 "밝은 색상 카드형 UI 사용" 등 구체적 지시어로 교정 제안

4. **사이버 보안 진단 가드라인 & 개인정보(PII) 스캐너**
   - **위험 행위 감지**: 해킹, 시스템 우회, 시험 부정, 허위조작 뉴스, 괴롭힘 및 보복성 글 탐지
   - **개인정보 감지**: 실명, 이메일, 폰번호, 주소, 금융 계좌, 주민등록번호, API 키 패턴 탐지 시 가상 정보 교체 조치 제공

5. **공격형 레드팀(Red Team) 모의 해킹 체험**
   - "이전 지시 무시(Injection)" 등 우회 공격 프롬프트를 시뮬레이션하고 방어 성공 여부와 근거 법률 학습

6. **모범 예시 라이브러리 & Good vs Bad 비교**
   - 발표 대본, 파이썬 코드 리팩토링 등 고효율 프롬프트 전후 비교 및 즉시 로드 제공

7. **성장 일지 아카이빙 & 저장 보관함**
   - LocalStorage를 통해 제작 완료된 프롬프트와 개선 기록(성공 소감, 전후 점수 차이)을 영구 보존 및 검색/수정

8. **발표 프레젠테이션 모드**
   - 설계 구조를 슬라이드 덱으로 전환하여 전체화면 프레젠테이션 및 인쇄 요약본 출력 지원

```text
    ├── index.css                # 글로벌 CSS 및 Tailwind v4 변수 테마
    ├── types.ts                 # 설계 컴포넌트 전역 타입 규격 정의
    ├── data.ts                  # 예시 템플릿, 페르소나 및 공격 데이터 프리셋
    ├── App.tsx                  # 실시간 컴파일 메인 레이아웃 및 스태퍼 상태 엔진
    └── components/
        ├── HelpManual.tsx       # 프롬프트 요소 사전 및 꿀팁 도움말
        ├── ImprovementLog.tsx   # 전후 점수 개선 타임라인 일기장
        ├── PresentationMode.tsx # 전체화면 발표 슬라이드 및 프린트 출력 엔진
        ├── PromptLibrary.tsx    # 모범 라이브러리 Good vs Bad 비교 뷰어
        ├── PromptOptimizer.tsx  # 가상 시뮬레이션, 조건 충돌 해결, 모호문 교정
        └── SavedPrompts.tsx     # 로컬 보관함 목록 조회, 검색, 필터링 및 복제
```

## ⚡ 로컬 실행 및 빌드 방법

### 1. 개발 서버 구동
독립 서버를 기동하여 핫 리로드 가능한 가상 브라우저에서 개발합니다. (Port는 기본 `3000`으로 바이딩됩니다)
```bash
npm run dev
```

### 2. 소스코드 결함 정적 검사 (Linter)
```bash
npm run lint
```

### 3. 배포용 빌드 컴파일
```bash
npm run build
```

## 🚀 GitHub Pages 배포 방법

본 플랫폼은 순수 **Client-Side SPA(단일 페이지 웹앱)**로서 어떠한 DB 서버나 Node 백엔드 없이 정적 호스팅이 가능합니다.

1. **Vite 설정 경로 수정**:
   - 만약 리포지토리 이름이 `https://github.com/username/repo-name` 형태라면, `vite.config.ts` 파일 내부 `defineConfig` 반환값에 `base: '/repo-name/'` 설정을 추가합니다.
2. **빌드 수행**:
   ```bash
   npm run build
   ```
   실행 결과물로 `dist/` 폴더 내에 최적화된 정적 HTML, JS, CSS 자원이 추출됩니다.
3. **GitHub Pages 활성화**:
   - GitHub 리포지토리의 **Settings > Pages** 메뉴로 이동합니다.
   - **Build and deployment > Source** 옵션을 `Deploy from a branch`로 선택합니다.
   - 브랜치를 `main` 또는 `gh-pages`로 지정한 뒤, 빌드 결과물인 `/docs` 혹은 `dist/` 폴더를 푸시하여 호스팅을 활성화합니다. (간편한 무중단 배포를 위해 `gh-pages` npm 패키지를 이용한 배포를 권장합니다)

## 🔒 개인정보 보호 및 사이버 안전 주의 사항

* **외부 API 독립성 보장**: 이 웹앱은 어떠한 상용 AI 서버나 개인 연락처 DB API와 연결되어 있지 않습니다. 모든 검출, 점수 계산, 레드팀 시뮬레이션은 어플리케이션 **내부의 규칙 기반(Rule-based Regex) 연산**으로만 즉각 처리되어 기밀 정보 유출에 전면 안전합니다.
* **실제 데이터 기입 불허**: AI 서비스에 질문을 보낼 때는 **실명, 집/학교 상세 주소, 주민등록번호, 계좌 정보, 비밀번호, 클라우드 API 키** 등은 절대 포함해서는 안 됩니다. 플랫폼 경고가 울릴 시 가상 정보(예: 홍길동) 치환 기능을 사용해 정보를 정화해 주세요.
* **사용자 사전 재검토 조항**: 본 안전성 필터는 규칙 기반 정밀 검사 시스템이나, 완벽한 판정을 보장하지는 못합니다. AI에 프롬프트를 입력하기 전, 사용자 스스로 도덕성과 위법 침해 여부를 한 번 더 숙고해 주시기 바랍니다.
